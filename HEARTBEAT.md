# HEARTBEAT.md - 定时任务检查与每日总结

## 🎯 设计原则

**每个定时任务都必须有对应的 Heartbeat 检查点**，确保：
1. 任务按时执行
2. 结果及时汇报
3. 异常主动发现

---

## ⏰ 定时任务清单

| 时间 | 任务名称 | 检查项 | 汇报内容 |
|------|----------|--------|----------|
| **10:00** | 待办提醒 | 检查 `memory/YYYY-MM-DD.md` 待办 | 发送今日待办列表 |
| **12:00** | OpenClaw 进展追踪 | 检查 `memory/openclaw-news-*.md` | 汇报最新版本信息 |
| **21:00** | 每日工作总结 | 检查 `memory/YYYY-MM-DD.md` 完整性 | 生成并发送工作日报 |

---

## 📋 10:00 待办提醒检查

**触发条件**: Heartbeat 在 10:00-11:00 之间执行

**检查清单**:
- [ ] 读取 `memory/YYYY-MM-DD.md` 中的待办
- [ ] 读取 `memory/persistent-todos.md` 中的持续待办
- [ ] 检查 `MEMORY.md` 中的待跟进事项

**执行动作**:
```
📅 今日待办 (YYYY-MM-DD)

1. [待办内容]
   来源: [引用的原始消息]

📌 持续跟进
• [持续待办内容]
```

**提醒后操作**:
- [ ] 从 `MEMORY.md` 中删除该待办
- [ ] 在 `memory/YYYY-MM-DD.md` 中标记为 ✅ 已完成
- [ ] 避免重复提醒

---

## 📋 12:00 OpenClaw 进展追踪检查

**触发条件**: Heartbeat 在 12:00-13:00 之间执行

**检查清单**:
- [ ] 检查 `memory/openclaw-news-YYYY-MM-DD.md` 是否存在
- [ ] 检查文件内容是否包含今日数据

**执行动作**:
```
📰 OpenClaw 每日进展报告 (YYYY-MM-DD)

最新版本：[版本号]
主要更新：[3-5条关键点]

📄 完整报告：[文件路径]
```

**异常处理**:
- [ ] 如报告未生成，检查 cron 日志
- [ ] 如内容为空，手动执行脚本重新生成

---

## 📋 21:00 每日工作总结检查

**触发条件**: Heartbeat 在 21:00-22:00 之间执行

**检查清单**:
- [ ] 检查 `memory/YYYY-MM-DD.md` 是否存在
- [ ] 检查日志是否完整（上午/下午/晚上）
- [ ] 检查 git 提交记录
- [ ] 统计今日完成项

**执行动作**:
```
📅 2026-XX-XX 工作日报

📊 今日统计
• Git 提交：X 次
• 新增文件：X 个
• 完成任务：X 项

🏆 关键成果
1. [成果1]
2. [成果2]

📌 持续待办
• [待办内容]

📁 今日新增
• [文件列表]
```

**完成后操作**:
- [ ] 提交更新后的工作日志到 GitHub
- [ ] 确认推送成功

---

## 🔄 Heartbeat 执行流程

```
收到 Heartbeat 触发
    ↓
检查当前时间 → 确定检查类型（10:00/12:00/21:00）
    ↓
执行对应检查清单
    ↓
生成汇报内容
    ↓
发送消息给用户
    ↓
执行后续操作（标记完成/提交GitHub等）
    ↓
记录检查状态到日志
```

```json
  "pendingReports": []

```

## ⚠️ 异常处理

**情况 1：定时任务未执行**
- 检查 crontab：`crontab -l`
- 检查脚本权限：`ls -la scripts/`
- 检查日志：`cat ~/.openclaw/*.log`

**情况 2：报告已生成但未汇报**
- 立即发送汇报消息
- 更新状态文件

**情况 3：Heartbeat 未触发**
- 用户手动发送消息触发
- 检查 OpenClaw Gateway 状态
