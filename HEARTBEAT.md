# HEARTBEAT.md - 定时任务检查与每日总结

## 🎯 设计原则

**每个定时任务都必须有对应的 Heartbeat 检查点**，确保：
1. 任务按时执行
2. 结果及时汇报
3. 异常主动发现

---

## ⏰ 定时任务清单

| 执行时间 | 检测时间 | 任务名称 | 检查项 | 汇报内容 |
|----------|----------|----------|--------|----------|
| **10:00** | **10:05** | 待办提醒 | 检查 `memory/YYYY-MM-DD.md` 待办 | 发送今日待办列表 |
| **12:00** | **12:05** | OpenClaw 进展追踪 | 检查 `memory/openclaw-news-*.md` | 汇报最新版本信息 |
| **21:00** | **21:05** | 每日工作总结 | 检查 `memory/YYYY-MM-DD.md` 完整性 | 生成并发送工作日报 |

> ⚠️ **重要**: 检测时间比执行时间晚 5 分钟，避免竞态条件

---

## 📋 10:00 待办提醒检查

**触发条件**: Heartbeat 在 10:00-11:00 之间执行

**检查清单**:
- [ ] 读取 `memory/YYYY-MM-DD.md` 中的待办
- [ ] 读取 `memory/persistent-todos.md` 中的持续待办
- [ ] 检查 `MEMORY.md` 中的待跟进事项

**执行动作**:
```
📅 今日待办 (YYYY-MM-DD)

1. [待办内容]
   来源: [引用的原始消息]

📌 持续跟进
• [持续待办内容]
```

**提醒后操作**:
- [ ] 从 `MEMORY.md` 中删除该待办
- [ ] 在 `memory/YYYY-MM-DD.md` 中标记为 ✅ 已完成
- [ ] 避免重复提醒

## 📋 12:00 OpenClaw 进展追踪检查

**触发条件**: Heartbeat 在 12:00-13:00 之间执行

**检查清单**:
- [ ] 检查 `memory/openclaw-news-YYYY-MM-DD.md` 是否存在
- [ ] 检查文件内容是否包含今日数据

**执行动作**:
```

最新版本：[版本号]

```

**异常处理**:
- [ ] 如报告未生成，检查 cron 日志
- [ ] 如内容为空，手动执行脚本重新生成

**触发条件**: Heartbeat 在 21:05-21:30 之间执行

## 🔄 Heartbeat 执行流程

```
收到 Heartbeat 触发
    ↓
检查当前时间 → 确定检查类型（10:05/12:05/21:05）
    ↓
检查是否在检测窗口期内（执行时间 + 5分钟）
    ↓
等待策略：如任务可能未完成，延迟检查
    ↓
执行对应检查清单
    ↓
生成汇报内容
    ↓
发送消息给用户
    ↓
执行后续操作（标记完成/提交GitHub等）
    ↓
记录检查状态到日志
```

## ⏱️ 竞态条件处理策略

### 问题：任务执行中 vs 心跳检测

**场景**: 任务 10:00 开始，心跳 10:05 检测，但任务需要 10 分钟完成

### 解决方案

#### 方案 1：时间错开（已采用）
- 任务执行：整点（10:00）
- 心跳检测：整点后 5 分钟（10:05）
- 适用于：执行时间 < 5 分钟的任务

#### 方案 2：状态文件 + 超时重试

脚本生成状态文件：
```bash
# 任务开始时
echo '{"status": "running", "startedAt": "'$(date -Iseconds)'"}' \
  > ~/.openclaw/task-status.json

# 任务完成时
echo '{"status": "completed", "completedAt": "'$(date -Iseconds)'"}' \
  > ~/.openclaw/task-status.json
```

Heartbeat 检测逻辑：
```
检查状态文件
    ↓
status = "completed"? → 立即汇报
    ↓
status = "running"? → 等待 2 分钟后重试
    ↓
重试 3 次仍未完成 → 标记异常，发送警告
```

#### 方案 3：宽容时间窗口

检测时不严格要求文件已生成，而是：
- 检查昨日文件是否存在（确认任务曾正常运行）
- 检查当前时间是否在合理范围内
- 如文件未生成，延迟 10 分钟后再次检查

---

## 📋 检测时间窗口定义

| 任务 | 执行时间 | 最早检测 | 最晚检测 | 超时警告 |
|------|----------|----------|----------|----------|
| 待办提醒 | 10:00 | 10:05 | 10:30 | 10:35 |
| OpenClaw 进展 | 12:00 | 12:05 | 12:30 | 12:35 |
| 工作日报 | 21:00 | 21:05 | 21:30 | 21:35 |

**规则**:
- 在 [最早, 最晚] 窗口内检测到 → 正常汇报
- 超过最晚检测时间 → 标记为延迟，汇报时注明
- 超过超时警告时间 → 发送异常警告

## ⚠️ 异常处理

**情况 1：定时任务未执行**
- 检查 crontab：`crontab -l`
- 检查脚本权限：`ls -la scripts/`
- 检查日志：`cat ~/.openclaw/*.log`

**情况 2：报告已生成但未汇报**
- 立即发送汇报消息
- 更新状态文件

**情况 3：Heartbeat 未触发**
- 用户手动发送消息触发
- 检查 OpenClaw Gateway 状态
